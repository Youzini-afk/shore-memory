## 1. 核心视觉设计规范

### 1.1 像素基础与渲染要求

- **全局渲染设置**：必须在全局 CSS 中开启 `image-rendering: pixelated;`，确保像素边缘不被平滑处理。
- **栅格对齐**：所有 UI 边框、间距应遵循 2px 或 4px 的整数倍，避免出现模糊渲染。
- **字体方案**：
  - 推荐字体：`Zpix`（首选）或 `DotGothic16`。
  - 渲染规则：关闭字体平滑（Anti-aliasing），保持笔画边缘的颗粒感。

---

### 1.2 配色系统

基于「萌动链接」设计稿提取的标准配色方案，所有组件应严格遵循此色板：

| 色彩名称 | 色值 | 用途 |
|----------|------|------|
| **天空蓝** | `#A7D8F0` | 全局大面积背景、滚动条 |
| **樱花粉**（主色） | `#F9A8D4` | 核心操作按钮、高亮状态、重点标识 |
| **梦幻紫**（辅助色） | `#C084FC` | 次要标题、修饰图标、渐变过渡 |
| **香草黄**（提示色） | `#FDE047` | 系统通知、警告提示、装饰性星形元素 |
| **可可褐**（描边色） | `#2D1B1E` | 全局统一描边，替换纯黑或深灰边框 |
| **云朵白**（面板色） | `#FFFFFF` | 卡片与面板背景，建议使用 80%–90% 不透明度 |

---

### 1.3 纹理与材质规范

- **背景渐变**：背景采用从「天空蓝（`#A7D8F0`）」到「淡紫色（`#E8D5FF`）」的 45° 线性渐变。
- **云朵蒙版**：主容器需叠加低透明度的像素云朵背景层（CSS 类名：`pixel-clouds-bg`）。
- **高光与阴影规则**：
  - 按钮 / 卡片增加 2px 白色像素内阴影（Top/Left 方向）模拟光泽。
  - 增加 2px 深粉色或深褐色外阴影（Bottom/Right 方向）模拟立体感。

## 2. 组件重构规范

### 2.1 像素边框

**标准 CSS 类定义**

| 类名 | 描述 |
|------|------|
| `.pixel-border-moe` | 使用「可可褐（`#2D1B1E`）」作为主描边颜色，2px 实线边框 |
| `.pixel-card-moe` | 白色背景 + 可可褐边框 + 粉色柔和投影 |

**Minecraft 风格三层边框规范**

| 层级 | 方向 | 规格 |
|------|------|------|
| **外框** | 全向 | 2px 可可褐实线 |
| **内侧高光** | Top / Left | 2px 纯白，透明度 0.3 |
| **内侧阴影** | Bottom / Right | 2px 深色，透明度 0.1 |

---

### 2.2 交互反馈规范

所有可交互组件必须实现以下三种反馈效果：

| 效果名称 | CSS 类名 | 实现方式 |
|----------|----------|----------|
| **悬停位移（Lift）** | `.pixel-hover-lift` | 悬停时 `translateY(-2px)` |
| **按压感（Press）** | `.press-effect` | 激活时 `translate(1px, 1px)` 并加深背景色 |
| **步进过渡（Stepped Animation）** | — | 使用 `transition-timing-function: steps(4)` 使色彩与位置变化具有帧动画感 |

## 3. 页面具体适配方案

### 3.1 启动器（LauncherView）

- **背景**：引入像素云朵与爱心漂浮动画。
- **进度条**：改为樱花粉填充，带有步进式像素条纹动画效果。

---

### 3.2 标题栏（CustomTitleBar）

- **主题**：整体改为粉色调，配合白色图标。
- **关闭按钮**：Hover 效果改为深褐色背景、白色「×」图标。

---

### 3.3 资源管理器（FileExplorer）

- **边框**：暗色模式下仍统一使用带透明度的「可可褐」描边，保持视觉一致性。
- **图标配色**：根据文件类型赋予对应主题色彩。

| 文件类型 | 建议配色 |
|----------|----------|
| `.py` | 梦幻紫（`#C084FC`） |
| `.json` | 香草黄（`#FDE047`） |
| 其他 | 天空蓝或云朵白 |

---

## 4. 动画风格规范

所有动画效果必须符合像素帧动画的整体基调，避免使用贝塞尔曲线类的平滑过渡。

| 动画名称 | 适用场景 | 实现要点 |
|----------|----------|----------|
| **Jelly（弹性）** | 核心操作按钮点击 | 极小幅度 `scale` 抖动，步进帧 |
| **Twinkle（闪烁）** | 装饰性星形元素 | 香草黄星星随机透明度变化 |
| **Pixel-Float（悬浮）** | 指示器、浮动图标 | 缓慢上下位移，4px 步进 |

---

> **一句话总结**：
> 所有视觉实现必须以「像素感」为核心约束，配色、边框、动画均应遵循本规范。任何组件新增或改动都应首先对照本文档校验一致性。
